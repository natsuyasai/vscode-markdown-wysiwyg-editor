# 要求明確化（フェーズ1）

実装やプラン策定に進む前に、要求を曖昧なまま進めないために要求内容を明確化し、ユーザーと合意する。

## 進め方

1. **要求の理解**: ユーザーの要求と関連コードを読み、何を実現したいのかを把握する。広範囲の調査が必要なら `Explore` / `general-purpose` サブエージェントで既存実装・パターンを調べる。
2. **曖昧点・不足の洗い出し**: 下記チェックリストで欠けている情報や複数解釈できる箇所を特定する。
3. **質問**: 不明点は `AskUserQuestion` で確認する。仕様や指示に疑問があれば作業を中断して質問する（CLAUDE.md基本ルール）。
4. **要求の確定**: 合意した要求を箇条書きで言語化し、ユーザーに認識合わせをする。

## 明確化チェックリスト

- [ ] 解決したい課題・背景（なぜやるのか）
- [ ] 対象範囲（どの画面・コンポーネント・機能か）
- [ ] 期待する振る舞い（入力・出力・エッジケース）
- [ ] 非対象（やらないこと）の明示
- [ ] 制約（既存仕様・パフォーマンス・互換性・UI/テーマ）
- [ ] 完了の判断基準（何が満たされれば完了か）

## 完了条件

- 上記チェックリストの不明点が解消されている
- 確定した要求がユーザーと合意されている

合意できたらフェーズ2（`implementation-planning`）へ進む。

## 禁止事項

- 大きな前提をユーザーに確認せず推測で埋めること
- 要求が曖昧なままプラン策定・実装に進むこと
