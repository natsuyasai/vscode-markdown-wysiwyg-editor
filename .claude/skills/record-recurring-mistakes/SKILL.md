# 再発ミスのCLAUDE.md追記（横断ルール）

アプリ開発フローの全フェーズを通じて、**同様のミスの指摘が複数回（2回以上）発生したら、必ず `CLAUDE.md` に再発防止ルールを追記する。**

レビュー指摘、Lint/型エラーの繰り返し、規約違反の再発などが対象。「次から気をつける」で終わらせず、ドキュメント化して恒久対策にする。

## 進め方

1. **検知**: 同種の指摘・修正が2回以上出たことに気づく（同一セッション内・複数セッションまたぎ問わない）。
2. **一般化**: 個別事象ではなく、再発を防ぐ一般的なルールとして言語化する。
3. **追記**: `CLAUDE.md` の適切な箇所（基本ルール／コマンド／規約など）に、具体的な指示として追記する。
4. **共有**: 追記内容をユーザーに伝える。

## 追記の書き方

- 何を・なぜ・どうするかを具体的に。再現条件と正しい対応をセットで書く。
- 悪い例: 「import順に注意する」
- 良い例: 「import順は外部パッケージ → `@message`/`@` エイリアス → 相対パスの順。グループ間の空行は入れない（`newlines-between: never`）」

## 判断の境界

- **CLAUDE.mdに追記する**: プロジェクト全体に効く規約・手順・繰り返す判断ミス
- **スキルを修正/新規作成する**: 手順が複雑で独立したワークフローになるもの（`writing-skills` 参照）
- **メモリに記録する**: コードからは読み取れない非自明な背景・経緯

## 禁止事項

- 同じ指摘が複数回出ているのに口頭の反省だけで済ませること
- 個別事象そのままで、一般化せずに追記すること
